1. CREATE DATABASE

Penjelasan: CREATE DATABASE digunakan untuk membuat sebuah database baru yang berfungsi untuk menampung dan mengelola tabel serta data.

CREATE DATABASE db_sekolah;

Contoh: Database bernama db_sekolah akan dibuat untuk menyimpan semua data yang berkaitan dengan sekolah.

2. CREATE TABLE

Penjelasan: CREATE TABLE digunakan untuk mendefinisikan tabel baru dalam database, lengkap dengan kolom-kolom yang diperlukan.

CREATE TABLE siswa (...);

Contoh: Tabel siswa akan berisi kolom untuk ID, NIS, Nama, dan informasi lainnya.

3. AUTO_INCREMENT

Penjelasan: AUTO_INCREMENT adalah atribut yang digunakan untuk kolom bertipe data numerik, seperti INT, untuk menghasilkan nilai unik secara otomatis setiap kali baris baru ditambahkan.

id_siswa INT AUTO_INCREMENT PRIMARY KEY

Contoh: Anda tidak perlu memasukkan nilai untuk id_siswa secara manual, karena nilainya akan bertambah secara otomatis setiap kali data siswa baru dimasukkan.

Manfaat: Menghindari kesalahan pengisian nilai ID yang duplikat dan mempermudah pengisian data tanpa harus mengatur ID secara manual.

4. FOREIGN KEY

Penjelasan: FOREIGN KEY digunakan untuk mendefinisikan relasi antar tabel, di mana sebuah kolom di satu tabel mengacu pada kolom di tabel lain untuk menjaga konsistensi data.

FOREIGN KEY (id_siswa) REFERENCES siswa(id_siswa)

Contoh: Kolom id_siswa pada tabel siswa_kelas mengacu pada kolom id_siswa di tabel siswa. Setiap data yang dimasukkan ke dalam tabel siswa_kelas harus memiliki id_siswa yang valid dari tabel siswa.

Manfaat: Memastikan data yang dimasukkan di tabel terkait (misalnya, tabel siswa_kelas) selalu terhubung dan valid dengan data di tabel lain.

5. ENUM

Penjelasan: ENUM adalah tipe data yang membatasi nilai kolom untuk hanya menerima satu dari beberapa nilai yang telah ditentukan.

jenis_kelamin ENUM('L','P')

Contoh: Kolom jenis_kelamin hanya dapat berisi nilai 'L' untuk Laki-laki atau 'P' untuk Perempuan. Dengan ini, kita bisa membatasi input agar hanya nilai yang valid yang dapat dimasukkan.

Manfaat: Menjaga konsistensi data dengan membatasi input hanya pada nilai yang telah ditentukan, dan menghindari kesalahan penulisan.

6. YEAR

Penjelasan: YEAR digunakan untuk menyimpan data yang hanya berupa tahun dalam format 4 digit.

tahun_masuk YEAR

Contoh: Kolom tahun_masuk menyimpan tahun masuk siswa (misalnya 2023), yang hanya menerima angka 4 digit.

Manfaat: Memudahkan pengelolaan data tahun, karena hanya menerima input tahun yang valid dalam format 4 digit.

7. INSERT INTO

Penjelasan: INSERT INTO digunakan untuk menambahkan data baru ke dalam tabel dengan menyebutkan nama kolom dan nilai yang ingin dimasukkan.

INSERT INTO siswa (nis, nama, jenis_kelamin, tanggal_lahir, alamat, tahun_masuk) VALUES ('S001', 'Ahmad Rizky', 'L', '2008-01-12', 'Jakarta', 2023);

Contoh: Perintah ini digunakan untuk memasukkan data siswa baru ke dalam tabel siswa. Nilai-nilai yang dimasukkan akan sesuai dengan urutan kolom yang telah didefinisikan.

Manfaat: Mempermudah penambahan data ke dalam tabel yang sudah ada, memastikan bahwa data yang dimasukkan sesuai dengan struktur tabel yang telah ditentukan.

8. NULL & NOT NULL

Penjelasan: NULL berarti tidak ada nilai yang dimasukkan ke dalam kolom, sedangkan NOT NULL memastikan bahwa kolom tersebut wajib diisi.

jenis_kelamin ENUM('L','P') NOT NULL

Contoh: Kolom jenis_kelamin harus diisi, sehingga menggunakan NOT NULL untuk memastikan tidak ada data kosong.

Manfaat: Memastikan bahwa kolom yang diberikan NOT NULL selalu memiliki data, dan menghindari penyimpanan data yang tidak lengkap.

9. CHECK CONSTRAINT

Penjelasan: CHECK digunakan untuk membatasi nilai yang dapat dimasukkan ke dalam kolom berdasarkan kondisi tertentu.

umur_pemohon INT CHECK (umur_pemohon >= 17)

Contoh: Kolom umur_pemohon hanya akan menerima nilai lebih besar atau sama dengan 17.

Manfaat: Menjaga integritas data dengan memastikan nilai yang dimasukkan memenuhi kriteria yang telah ditentukan.

10. UNIQUE CONSTRAINT

Penjelasan: UNIQUE memastikan bahwa setiap nilai dalam kolom bersifat unik, tidak ada duplikasi.

nis VARCHAR(20) UNIQUE

Contoh: Kolom nis tidak dapat memiliki dua nilai yang sama di seluruh tabel.

Manfaat: Menghindari duplikasi data pada kolom yang harus unik, seperti nomor induk siswa (NIS) atau email.
